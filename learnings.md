## Temas aprendidos

- Backtracking.
- Complejidad algorítmica.
- Doble punteros para recorrer listas y vectores.
- Estructuras Dummies.
- Grafos.
- Listas y matrices de adyacencia.
- Manipulación de arboles binarios representados en vectores.
- Maps.
- Permutación y eliminación de nodos en Singly Linked Lists.
- Queues.
- Sets.
- Stacks.
- Utilizar el tipo de dato correcto para evitar problemas de Overflow, más que nada en TypeScript (JavaScript).

## Consejos a la hora de encarar un Challenge

Antes de comenzar a codificar cualquier posible solución a un Challenge debo:

1. Plantear en una hoja como llegar del punto A al punto B utilizando pseudo-código, con el fin de plantear una solución candidata.
2. Controlar y validar las restricciones del problema.
3. Plantear y definir como debe actuar mi solución candidata frente a los casos de borde.
4. Corroborar el correcto funcionamiento de la solución candidata utilizando los casos de prueba proporcionados.
5. Codificar la solución candidata y ejecutar los tests.

> Si el Challenge es muy complejo, es recomendable dividirlo en partes más pequeñas para facilitar su resolución.

# Bases de datos

## Temas aprendidos

- Bucles `WHILE` para realizar iteraciones.
- `CASE` para realizar condicionales en una consulta.
- Cómo concatenar cadenas de texto con otros tipos de datos.
- Cómo crear variables.
- Cómo manejar fechas.
- Cómo obtener una porción de una cadena de texto.
- Cómo transformar tipos de datos.
- `DISTINCT` para eliminar duplicados en los resultados.
- Expresiones regulares para la búsqueda de patrones en cadenas de texto.
- Funciones de agregación.
- `GROUP BY` para realizar cálculos sobre conjuntos.
- `IF` para realizar condicionales.
- `JOIN` para combinar datos de varias tablas.
- `OVER` para realizar cálculos sobre conjuntos.
- `WITH` para crear Common Table Expressions (CTEs) y dividir un problema complejo en partes más pequeñas y manejables.

## Consejos a la hora de encarar un Challenge

- Recordar que en SQL Server las igualaciones son con el operador `=`.
- Recordar que en SQL Server si se quiere cortar un flotante a `n` decimales, se debe utilizar la función `DECIMAL`.
- Recordar que la palabra reservada `DISTINCT` se utiliza para eliminar duplicados en los resultados.
- Utilizar Common Table Expressions (CTEs) para dividir un problema complejo en partes más pequeñas y manejables.
- Utilizar expresiones regulares para las condiciones de búsqueda de patrones en cadenas de texto.
